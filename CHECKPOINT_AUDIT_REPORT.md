## How Compression Works

1. When a new message is sent, the system loads message history
2. If a previous checkpoint exists, messages are loaded starting from that checkpoint
3. Token count is calculated for loaded messages + instructions
4. If tokens exceed threshold (18,000), compression triggers:
   - All loaded messages are summarized using recency-weighted prompt
   - Recent messages get [RECENT] marker for detailed summarization
   - Older messages get [OLD] marker for brief summarization
   - Summary is stored on the trigger message (second-to-last)
   - Payload sent to LLM: [instructions, summary, latest_message]
5. Next request starts fresh from the new checkpoint
